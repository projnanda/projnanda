# Protocol Interoperability & Reference Implementations

One page for protocols, interoperability, and the current reference stack.

---

## Protocol Interoperability

NANDA assumes agents must speak across ecosystems. We abstract over:
- **MCP** (Model Context Protocol)
- **A2A** (Agent-to-Agent)
- **HTTPS** (backbone of the web)
- **NLWeb** (Microsoft)

The Agent SDK sits above these, while the Index + AgentFacts standardize identity and trust.

---

## Current Reference Implementations

### 1) NANDA Index
**[index.projectnanda.org](https://index.projectnanda.org)** — Lightweight, decentralized registry of agent entries (DNS-like). Forkable and federated.

- Agents claim names
- Endpoints/metadata stored as verifiable entries (REST API)
- Multiple index-compatible registries can coexist

[GitHub: Registry Spec](https://github.com/projnanda/nanda-index)

### 2) AgentFacts
**[list39.org](https://list39.org)** — Structured facts describing capabilities, provenance, uptime, hosting.

- Human-readable, machine-verifiable
- Natural language + capability claims + telemetry links (Cert39) + provenance logs
- Hosted independently from the registry to decouple identity and state

[GitHub: AgentFacts Format](https://github.com/projnanda/agentfacts-format)

### 3) Agent Picker SDK
**[GitHub: NANDA SDK](https://github.com/projnanda/adapter)** — Python tool to automate Internet of Agents servers.

- Server config, DNS, SSL, MCP servers, Anthropic integration
- Deploy multiple agents on one server with production-ready HTTPS endpoints

[GitHub: Agent SDK](https://github.com/projnanda/adapter)

### 4) Join39 Agent Platform
**[join39.org](https://join39.org)** — Platform to create/manage personal AI agents (digital twins) on NANDA.

- Create/deploy personal agents, generate AgentFacts, upload MCP servers
- Bootstrap adoption with easy onboarding

### 5) NANDA Agent Framework
**[GitHub: NANDA Agent Framework](https://github.com/projnanda/adapter)** — Make any agent persistent, discoverable, and interoperable.

- Multiple AI frameworks (LangChain, CrewAI, custom)
- Multi-protocol communication
- Global Index integration
- SSL/production ready

## Features

- **Multi-protocol**: MCP, A2A, HTTPS, NLWeb abstraction
- **Global index**: Automatic discovery via NANDA Index
- **Interoperability**: Bridges and adapters across ecosystems
- **Production-ready**: SSL, DNS automation, deploy multiple agents per host
- **Open-source**: Forkable registries, transparent specs, SDKs
